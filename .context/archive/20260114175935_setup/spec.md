# Track: setup

## 1. User Intent
Bootstrap the project by defining the Product Vision, Domain Model, and Technical Constraints.

## 2. Relevant Context
Greenfield (No existing context)

## 3. Requirements (EARS)
- The project context shall be documented in `.context/product.md`, including Vision, Target Audience, and Key Value Propositions.
- The domain boundaries, events, and ubiquitous language shall be documented in `.context/domain.md` (using Event Storming output).
- The technical architecture and constraints shall be documented in `.context/tech-stack.md`.
- Where specific compliance or security constraints exist, they shall be recorded in `decisions.md`.
